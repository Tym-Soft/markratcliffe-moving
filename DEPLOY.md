# Production deployment — GitHub Pages

The site is published from the `main` branch of [Tym-Soft/markratcliffemoving.co.uk](https://github.com/Tym-Soft/markratcliffemoving.co.uk) directly to GitHub Pages. Every `git push` deploys.

---

## One-time GitHub Pages setup

After the first push lands on `main`:

| | Step |
|---|---|
| ☐ | Go to **https://github.com/Tym-Soft/markratcliffemoving.co.uk/settings/pages** |
| ☐ | Under **Source**: select `Deploy from a branch` |
| ☐ | Branch: **`main`** · Folder: **`/ (root)`** · Save |
| ☐ | Wait ~1 minute, then refresh — you should see "Your site is live at https://www.markratcliffemoving.co.uk" once DNS resolves |
| ☐ | Tick **Enforce HTTPS** once the SSL cert is issued (5–60 min after DNS propagates) |

The `CNAME` file at the repo root tells GitHub to serve from `www.markratcliffemoving.co.uk`. Do not change it manually inside the Pages UI — keep the file as the source of truth.

## One-time DNS setup

You need two DNS records at your domain registrar. GitHub Pages requires both A records (for the apex `markratcliffemoving.co.uk`) and a CNAME (for the `www.` subdomain).

### At the registrar that controls `markratcliffemoving.co.uk`

**A records for the apex** (so `markratcliffemoving.co.uk` redirects to `www.`):

```
Type   Host   Value             TTL
A      @      185.199.108.153   3600
A      @      185.199.109.153   3600
A      @      185.199.110.153   3600
A      @      185.199.111.153   3600
```

**CNAME for the `www` subdomain**:

```
Type    Host   Value                                       TTL
CNAME   www    tym-soft.github.io                          3600
```

> **Note:** if your old host has the domain pointing somewhere else, you must change these records there before GH Pages will serve. Existing A/CNAME records for `@` and `www` must be deleted or replaced.

DNS propagation takes anywhere from minutes to a few hours. Check with `dig +short www.markratcliffemoving.co.uk` — you should see `tym-soft.github.io` or a `185.199.x.x` IP.

## Day-to-day publishing workflow

```bash
# Make your edits locally
cd ~/Desktop/markratcliffemoving.co.uk

# Stage + commit
git add .
git commit -m "Update homepage hero copy"

# Push — GH Pages rebuilds within ~30 seconds
git push
```

That's the whole flow. Check the deploy status at the **Actions** tab of the repo (look for "pages build and deployment").

## What the local-only admin panel does

The PHP admin (`admin-portal-xK7p9q/`) is **excluded from git** via `.gitignore`. It runs locally:

```bash
cd ~/Desktop/markratcliffemoving.co.uk
php -S localhost:8000
# Visit http://localhost:8000/admin-portal-xK7p9q/login.php
```

Use it to edit pages through the admin's GUI. When you save a page, it rewrites the corresponding HTML file (and regenerates `sitemap.xml`). After editing, commit + push as above to deploy.

If you don't want to use the admin, you can edit the HTML files directly with any editor — they're plain HTML.

## Troubleshooting

| Symptom | Likely cause / fix |
|---|---|
| Push succeeds but site doesn't update | Check **Actions** tab → "pages build and deployment" — if red, click for the build log |
| "404 — page not found" at root | Pages may not be enabled yet. Settings → Pages, set source to `main` / `(root)` |
| Site loads on `tym-soft.github.io/...` but not `www.markratcliffemoving.co.uk` | DNS hasn't propagated yet. Run `dig +short www.markratcliffemoving.co.uk`. If it doesn't return GitHub IPs, your DNS isn't pointing yet — fix at registrar |
| HTTPS shows "Not secure" | Wait for the cert to be issued (up to 60 min after DNS resolves). Then tick **Enforce HTTPS** in Pages settings |
| Pages publishes the repo's `README.md` instead of `index.html` | `.nojekyll` is missing from the root. The file is empty but must exist |
| Old `/testimonials.html` URL serves a redirect stub but Google still indexes the old URL | Submit `/sitemap.xml` to Search Console and request reindexing |

That's it. Repo + GH Pages + DNS = live site.
